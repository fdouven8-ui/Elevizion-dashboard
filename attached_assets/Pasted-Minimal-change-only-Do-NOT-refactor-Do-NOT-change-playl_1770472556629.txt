Context / Problem:
- Media 29860650 exists in Yodeck with media_origin.source = "local" and status = "finished".
- But its arguments still contain:
  - download_from_url: "https://dsbackend.s3.amazonaws.com/..."
  - play_from_url: "https://dsbackend.s3.amazonaws.com/..."
  - buffering: true
- This causes 5–10s “geen netwerk” when the ad starts (player treats it like URL streaming).
- We created a repair endpoint and called:
  POST /api/admin/yodeck/media/29860650/repair-local
  which returned HTTP 400 with Yodeck error err_1003: "Invalid field name or value".
- Root cause: Yodeck PATCH does NOT accept null values for these fields. We must use empty strings or omit fields.

Goal:
For LOCAL media uploads (and the repair endpoint), clear URL playback fields without re-upload:
- arguments.download_from_url should become "" (empty string)
- arguments.play_from_url should become "" (empty string)
- arguments.buffering should become false
This should stop “geen netwerk” at start for current and future ads.

A) Normalization: make it best-effort and cheap
- Current ffmpeg command re-encodes (libx264) and gets "Killed".
- Change normalization to prefer a fast stream-copy when possible:
  ffmpeg -y -i INPUT -c:v copy -movflags +faststart -an OUTPUT
- If stream-copy fails, fall back to a very light re-encode:
  ffmpeg -y -i INPUT -c:v libx264 -preset ultrafast -crf 28 -pix_fmt yuv420p -movflags +faststart -an OUTPUT
- If BOTH fail (or process killed), do NOT set readinessStatus=REJECTED. Instead:
  - set normalizationError but continue publish using ORIGINAL storagePath as the upload source.
  - Log: [Normalize] FAILED_BUT_CONTINUING reason=...
This removes "Geen Yodeck media ID" purely due to normalization.

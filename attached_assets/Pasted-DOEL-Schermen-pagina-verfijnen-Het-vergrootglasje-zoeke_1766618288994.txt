CONTEXT
Project: React + TypeScript + Vite.
Pagina: Schermen (lijstweergave met filters en tabel).
Gebruik lucide-react icons (zoals elders). Tailwind classes worden gebruikt.

TAKEN

1) ZOEKEN TERUG (vergrootglas)
- Voeg in de Filters/toolbar een Search input toe met een vergrootglas-icoon links in het veld.
- Placeholder: “Zoek op scherm, locatie, plaats, ID…”
- Zoek moet client-side filteren op:
  - schermnaam (Yodeck Naam / Scherm)
  - Screen ID (EVZ-xxx als je die hebt) en/of Yodeck code (YDK-xxxxx)
  - plaats
  - locatie/bedrijf
  - content titel (als zichtbaar in de tabel)
- Zoek moet werken samen met bestaande filters (plaats, locatie, online/offline, plaatsingen min/max).
- Debounce 150–250ms (zodat typen soepel blijft).

2) “OPEN” MEER IN HET ZICHT (primary actie)
- Maak “Open” een duidelijke primary knop:
  - Groter click-target (min height 36–40px)
  - Plaats de knop meer “in het zicht”:
    Optie A (voorkeur): verplaats de Actie-kolom naar meer links (net na Status of zelfs direct na Scherm).
    Optie B: laat kolommen staan, maar maak de knop visueel primary (filled/green) + voeg een “↗”/external icon.
- Bonus (als makkelijk): maak de hele rij klikbaar om te openen (cursor-pointer), behalve wanneer je op filters/checkboxes klikt.
- Zorg dat accessibility ok is (button blijft button, row-click gebruikt onClick met role/link of nav).

3) UI DETAILS
- In de toolbar rechts (waar nu “2/2” + view icons staan): laat de zoekinput NIET daar verdwijnen; hij moet altijd zichtbaar zijn in de filterbalk (links/midden).
- Op mobiel: zoekinput full width boven de tabel, filters eronder (stack).
